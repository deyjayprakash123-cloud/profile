I don't build apps. I build **systems** — architectures that hold under pressure, scale without ceremony, and render at the threshold of hardware.

My engineering discipline spans three tightly coupled verticals: I design **real-time, interactive full-stack software systems** where every layer — from database schema to UI shader — is deliberate. I reach for **Three.js via React Three Fiber** when the interface demands spatial presence: immersive 3D scenes, procedural geometry, and WebGL-powered cinematic frontends that communicate information through environment, not just layout.

At the data layer, I don't just query — I model. I work with **complex relational and vector datasets through raw SQL**, designing normalized schemas that survive schema drift and serve analytics pipelines without bloating the response cycle. When the task crosses into intelligence, I reach into the **Scikit-Learn ML toolkit** — training, evaluating, and deploying custom classification and regression models with rigorous validation protocols.

For deployment infrastructure, I operate on a **Firebase + Google AI Studio + Vercel triad**: Firebase handles auth flows, Firestore's real-time event bus, and serverless function orchestration; Google AI Studio empowers my **local LLM engineering workflows using Gemma**, enabling on-device inference, prompt tuning, and RAG pipelines without cloud dependency; Vercel closes the loop on zero-downtime, edge-optimized delivery.

On mobile, I build **native Android applications in Kotlin** — threading-aware, Compose-driven UIs with Architecture Component compliance and lifecycle precision.

Every project under the **AEROX** umbrella is an assertion: that precision engineering and premium design are not mutually exclusive.
